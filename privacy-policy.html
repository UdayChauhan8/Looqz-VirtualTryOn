Privacy Policy

Last updated: July 14, 2026

Looqz Virtual Try-On ("the Extension") is a Chrome browser extension that allows users to virtually try on clothing items from any online shopping website using AI-powered image generation. This Privacy Policy explains what data the Extension accesses, how it is used, and how it is protected.

Summary: The Extension does not collect, store, or share any personal data on external servers controlled by us. All user data (API key and photo) is stored locally on your device.

1. Data We Access

The Extension accesses the following data solely to provide its virtual try-on functionality:

  • API Key: A Looqz API key you provide. This key is stored locally in your browser using chrome.storage.local and is sent directly to the Looqz API (looqz.in) to authenticate try-on requests.
  • User Photo: A photo you voluntarily upload. This photo is stored locally on your device using chrome.storage.local. When you initiate a try-on, the photo is temporarily uploaded to our image hosting server solely so the Looqz API can access it for processing. The uploaded file is automatically deleted within 5 minutes.
  • Clothing Image URL: The URL of a clothing image you select from a shopping website. This URL is passed to the Looqz API to generate the try-on result. It is not stored persistently.

2. Data We Do NOT Collect

The Extension does not collect, access, or transmit:

  • Personally identifiable information (name, email, address)
  • Browsing history or web activity
  • Cookies or authentication tokens from websites you visit
  • Keystroke data, mouse movements, or click patterns
  • Financial or payment information
  • Location data
  • Any data from websites you do not actively interact with through the Extension

3. How Data Is Used

All data accessed by the Extension is used exclusively for the single purpose of generating virtual try-on images:

  • Your API key authenticates requests to the Looqz API.
  • Your photo and the selected clothing image are sent to the Looqz API to generate a virtual try-on result.
  • Your credit balance is retrieved from Looqz to display remaining credits in the sidebar.

4. Data Storage

  • Local storage only: Your API key and uploaded photo are stored using chrome.storage.local, which keeps data entirely on your device. This data is never synced to any cloud service.
  • Temporary image hosting: When you initiate a try-on, your photo is temporarily uploaded to our server (looqz-backend-q05q.onrender.com) to provide the Looqz API with an accessible URL. These temporary files are automatically deleted within 5 minutes by an automated background process.

5. Third-Party Services

The Extension interacts with the following third-party service:

  • Looqz API (looqz.in): The AI-powered virtual try-on service. Your photo and selected clothing image are sent to this service to generate the try-on result. Please refer to Looqz's privacy policy for their data handling practices.

6. Data Sharing

We do not:

  • Sell or transfer user data to third parties
  • Use user data for advertising, analytics, or profiling
  • Use or transfer user data for purposes unrelated to the Extension's core try-on functionality
  • Use or transfer user data to determine creditworthiness or for lending purposes

7. Data Security

We implement the following security measures:

  • All API communication uses HTTPS encryption
  • CORS restrictions limit backend access to the Extension only
  • Temporary files are protected against directory traversal attacks
  • File uploads are limited to 10 MB and restricted to image formats
  • Rate limiting is enforced on all endpoints

8. User Control

You have full control over your data:

  • Reset Extension: Use the "Reset Extension" option in the Extension's settings panel to delete all locally stored data (API key and photo).
  • Change API Key: Use the "Change" option to remove and replace your stored API key.
  • Uninstall: Uninstalling the Extension automatically removes all locally stored data.

9. Permissions

The Extension requests the following Chrome permissions, each essential to its functionality:

  • storage: To save your API key and photo locally on your device.
  • activeTab: To access the current tab when you click the Extension icon, enabling sidebar injection and image selection.
  • scripting: To programmatically inject the sidebar UI into the active tab on demand.
  • Host permissions (<all_urls>): To allow the Extension to work on any e-commerce website, since there is no fixed list of supported shopping sites.

10. Children's Privacy

The Extension is not directed at children under the age of 13, and we do not knowingly collect data from children.

11. Changes to This Policy

We may update this Privacy Policy from time to time. Any changes will be reflected by updating the "Last updated" date at the top of this page. Continued use of the Extension after changes constitutes acceptance of the revised policy.
